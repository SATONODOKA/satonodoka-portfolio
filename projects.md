プロジェクト1
工務店 × 職人 マッチング支援アプリ（LINE連携）
https://shokuninboshu.netlify.app/
概要
人手不足の工務店が個別に職人を確保できるようにすること、また個人事業主の職人が仕事が途切れにくい状態を作ることを目的に開発した、工務店向けアプリケーション。
起業コンテスト向けのプロトタイプとして作成。

電話や紹介に依存しがちな案件獲得・依頼のやり方を見直し、LINEを活用した効率的なマッチングを目指した。

課題

工務店側：

慢性的な人手不足

案件ごとに職人を探す負荷が高い

職人側：

仕事が不定期で、案件が途切れやすい

新しい元請けと接点を持ちづらい

アプローチ

職人をLINE友だちとして登録

案件発生時に、近隣・条件に合う職人へ一斉配信

LINE上でのやり取りをそのまま業務フローに組み込む設計

実装内容

実際のLINE Messaging APIと連携し、案件配信・返信を実現

職人・案件・メッセージ履歴をデータベースで管理

工務店側がWeb画面から配信・状況確認できる構成

成果・ポイント

電話や個別連絡に依存しない案件配信が可能

やり取りの履歴を可視化し、属人化を防止

実運用を想定したデータ設計とAPI連携まで実装

技術スタック

フロントエンド：React（TypeScript併用、react@^19）

言語：TypeScript

ビルド／開発：Vite（@vitejs/plugin-react）

スタイリング：Tailwind CSS + PostCSS + Autoprefixer

型定義／開発依存：TypeScript ^5.9、@types/node / react / react-dom

プロジェクト2
法人向け 求職者情報検索・転記 RPA

概要
複数の社内システムを横断して行われていた、求職者情報の検索・書類取得・転記業務を完全自動化するために開発した法人向けRPAツール。

一見単純だが、個人情報を扱うため高い安全性と正確性が求められる業務を対象としている。

課題

CA・RAが複数システムを行き来しながら手作業で対応

転記ミス・確認漏れのリスクが常に存在

工数が大きく、業務コストが高止まりしていた

IEでしか動かないレガシーシステムが混在し、例外対応が多い

対象業務フロー

システム1（Kintoneベース）で求職者検索

IDを用いてシステム2（自社オリジナルDB）を検索

履歴書ファイルをダウンロード

内容を参照し、システム3（採用ATS）へ転記

アプローチ

各システムのUI操作を前提としたRPA設計

レガシー環境（IE依存・例外挙動）を想定した分岐処理

エラー発生時の安全停止・ログ取得を重視

実装内容

Python／Puppeteer／Playwright／SendKeys系ツールを組み合わせた自動化

検索 → ファイル取得 → 内容参照 → 転記までを一気通貫で制御

業務ルールに沿った例外処理を個別に実装

成果・ポイント

手作業を前提としていた業務を完全自動化

人的ミスの削減と処理時間の短縮

大きなコスト削減に寄与

レガシー環境を含めて運用可能な形で実装し切った点が最大の成果

技術スタック

フロントエンド：React（TypeScript併用、react@^19）

言語：TypeScript

ビルド／開発：Vite（@vitejs/plugin-react）

スタイリング：Tailwind CSS + PostCSS + Autoprefixer

自動化：Puppeteer / Playwright / SendKeys 系ツール

プロジェクト3
求人保存・整理 Chrome拡張 + 管理サイト
https://job-seiri.netlify.app/
概要
求人サイトを閲覧中に、Chrome拡張機能をワンクリックするだけで求人を保存し、自動でタグ付け・整理できる個人向けツール。

エージェント任せではなく、自分で調べながら転職活動を進めたい人を想定して作成。

課題

複数求人サイトを横断して見ると情報管理が煩雑

気になった求人を後で探し直す手間が発生

手動でのコピペ・メモ管理に時間がかかる

アプローチ

ブラウザ上で完結する操作感を重視

「保存する」という行為を最小操作にする

後から探しやすいよう、タグによる整理を自動化

実装内容

Chrome拡張から現在閲覧中の求人情報を取得

Webサイト側のデータベースに保存

求人内容に応じてタグを自動付与

保存済み求人を一覧・タグ別に管理可能

成果・ポイント

求人収集・整理にかかる手作業を削減

自分の軸で求人を比較・検討しやすくなった

個人利用を前提としたシンプルな運用設計

技術スタック

フロントエンド：HTML + JavaScript

言語：JavaScript / HTML / Windowsバッチ（.bat）

ビルド／開発：Node.js、npm

スタイリング：HTML / CSS（フレームワーク不使用）
