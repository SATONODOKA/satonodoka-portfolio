# プロジェクトページエラー修正プラン

## 問題の整理

### エラー1: Server ComponentでonErrorを使用
- **場所**: `app/work/[slug]/page.tsx` 182行目
- **問題**: Server Componentでイベントハンドラー（onError）を使用している
- **影響**: ビルドエラーまたはランタイムエラー

### エラー2: フォールバック機能が不要
- **問題**: ユーザーはフォールバックを不要としている
- **対応**: フォールバック機能を削除し、画像を確実に表示する

### エラー3: 画像ファイルが存在しない可能性
- **問題**: 画像ファイルが`public/images/`に存在しない
- **対応**: 画像を取得して配置するか、画像パスを確認する

## 修正ステップ

### ステップ1: app/work/[slug]/page.tsxのonErrorを削除
- Screensセクションの画像表示部分を修正
- onErrorハンドラーを削除
- 画像が存在しない場合は表示しない（エラーを出さない）

### ステップ2: ProjectImageコンポーネントからフォールバックを削除
- フォールバック機能を完全に削除
- 画像が読み込めない場合は何も表示しない

### ステップ3: 画像ファイルの確認と取得
- `public/images/`ディレクトリの内容を確認
- 必要な画像ファイルをリストアップ
- 画像取得方法を実装または手動配置を案内

### ステップ4: 画像表示の確実化
- Next.jsのImageコンポーネントを使用するか検討
- 画像パスの確認と修正

## 実装順序

1. **app/work/[slug]/page.tsxの修正**（最優先）
2. **ProjectImageコンポーネントの修正**
3. **画像ファイルの確認**
4. **画像表示の確実化**
